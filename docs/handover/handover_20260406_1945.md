## Work Completed

### 1. P0 Stability Fixes (commit `a0358b4`)
- **PushToTalkManager state machine race condition** — replaced all fire-and-forget `DispatchQueue.main.asyncAfter` with cancellable `DispatchWorkItem` properties. Max-duration timer (5 min) and cancelled→idle timer (0.3s) are now explicitly cancelled on every state transition. This was the primary root cause of "push-to-talk sometimes doesn't respond".
- **TextPolisher missing network timeout** — added `request.timeoutInterval = 10` to all 4 LLM API calls (Gemini, Qwen, OpenAI, Anthropic). Previously, a slow/unresponsive API would hang the UI at "Polishing..." for 60 seconds.
- **Event tap silent failure** — added `reEnableEventTap()` with 3 retry attempts (100ms apart). If all fail, user is notified via system notification. New `pushToTalkDidLoseEventTap()` delegate method.

### 2. P1 Stability Fixes (commit `641a760`)
- **AudioCapture thread safety** — added `os_unfair_lock` to protect `audioFile` access between the audio tap callback (real-time thread) and `stopRecording()` (main thread).
- **Clipboard restore race condition** — `TextInserter` now stores the pending restore `Task` and cancels it before each new paste cycle.
- **UI language unification** — changed all user-facing strings to Chinese across MenuBarController, RecordingOverlayPanel, NotificationManager, AppDelegate, PushToTalkManager (5 files).

### 3. Local LLM Provider (commit `70346c9`)
- Added `local` case to `PolishProvider` enum in AppConfig.
- Implemented `callLocal()` in TextPolisher — OpenAI-compatible endpoint, 30s timeout, no API key required.
- Added Settings UI section "本機 LLM" with endpoint URL and model name fields.
- Env vars: `LOCAL_LLM_ENDPOINT` (default: `http://localhost:11434/v1/chat/completions`), `LOCAL_LLM_MODEL`.
- Auto mode priority: Gemini > Qwen > OpenAI > Anthropic > Local.
- Tested with Ollama `gemma4:e2b` — polish quality good, latency ~13s (warm) on M4 16GB.

### 4. Student Distribution (commit `485364d` + GitHub Release `student-v1`)
- Built release `.app` bundle (482KB) and zipped `whisper-cli` (344KB).
- Created `install-student.sh` — one-liner installer that downloads app, whisper-cli, Whisper model (~1.5GB from HF), creates config and launcher script, optionally installs Ollama.
- Published GitHub Release: https://github.com/johnson-greate/Cantoflow/releases/tag/student-v1

### 5. Infrastructure Changes
- Modified `~/bin/cantoflow` global script — now sources `~/.cantoflow.env` for API keys, sets local LLM defaults, uses `--auto-replace`.
- Temporarily disabled LaunchAgent (renamed plist to `.disabled`) during testing, restored afterwards.

2. **CorrectionWatcher self-learning system** — audited and found to be partially broken (~6% success rate). Key bugs: flush() race condition, fragile anchor detection. Decision: leave as-is for now, don't advertise to students.

### Low Priority
6. **LaunchAgent conflicts** — when user runs `cantoflow` script, LaunchAgent's `KeepAlive` restarts old instances. The script now kills them, but a more robust solution would be to update the LaunchAgent plist itself.
7. **Gemma4:e2b performance** — 13s polish latency is too slow for daily use. Consider recommending smaller models or making polish truly async (paste raw immediately, replace when polish completes).

## Known Issues

- **CorrectionWatcher**: ~6% success rate. flush() race condition + fragile anchor detection. Not advertised to students.
- **LaunchAgent auto-restart**: `pkill cantoflow` triggers LaunchAgent restart. Workaround: `launchctl unload` first, or rename plist.
- **Whisper brand name recognition**: English brand names spoken in Cantonese (e.g., "Qwen" → "簽文") are not reliably corrected even with vocabulary hints.
- **16GB RAM constraint**: Running Ollama (gemma4:e2b ~2.5GB) + Claude Code + Chrome simultaneously causes memory pressure and swap usage. Recommend students close other apps when using local LLM.

---

## Key Decisions Made

1. **Qwen cloud as default, local LLM as demo option** — local model (13s latency) too slow for daily use, but valuable as a "future direction" demo for students.
2. **Pre-built distribution over source compilation** — students are not developers, shouldn't need Xcode.
3. **Chinese-only UI** — target audience is HK students, unified all strings to Chinese.
4. **Leave CorrectionWatcher as-is** — works occasionally but not reliable enough to fix now. Don't break it, don't promote it.
