## Changes Delivered

### 1. Qwen LLM Polish — Enabled & Working

**Problem:** `polish_status` was `not_run` on every recording.

**Root cause:** The development binary (`.build/release/cantoflow`) was launched directly from the terminal without the `QWEN_API_KEY` environment variable. The `/Applications/CantoFlow.app` launch script was not involved.

**Fix:**
- Created `~/.cantoflow.env` (permissions `600`) containing `QWEN_API_KEY`
- Created `scripts/run.sh` — wrapper that sources `~/.cantoflow.env` before launching the binary
- Also patched `/Applications/CantoFlow.app/Contents/MacOS/cantoflow` to source the env file for .app launches

**Verified:** `polish_status: ok`, `provider: qwen`, `polish` latency ~2–3.5s

### 2. Traditional Chinese Output — Fixed

  - Added rule 9 to system prompt: `必須以繁體中文輸出，將所有簡體字轉換為繁體字`

**Note:** Whisper `yue` prompt hints help but cannot guarantee 100% Traditional output. The LLM polish layer (rule 9) is the reliable backstop.

### 3. Terminal Double-Output Bug — Fixed

**Problem:** When dictating into macOS Terminal (e.g. Claude Code CLI), text appeared twice — raw text followed by polished text concatenated together.

**Root cause:** `fast_ime` auto-pastes raw text immediately, then sends `Cmd+Z` to undo it before pasting the polished version. In terminal apps, `Cmd+Z` does not undo text input, so both pastes accumulated.

**Fix:**
- `Sources/CantoFlowApp/Core/TextInserter.swift`
  - Added `terminalBundleIDs` set (Terminal, iTerm2, Warp, Kitty, VSCode, Alacritty, Hyper)
  - Added `isFrontmostAppTerminal() -> Bool`
- `Sources/CantoFlowApp/Core/STTPipeline.swift`
  - Detects terminal at pipeline start (before STT delay)
  - **Terminal path:** skips raw paste entirely; pastes polished text directly after LLM completes
  - **Regular app path:** unchanged (raw paste → Cmd+Z undo → paste polished)

**Telemetry confirmation:**
```
raw_status:     skipped_terminal
replace_status: terminal_paste
```

### 4. `cantoflow` Global Command

**Problem:** App could only be launched from the `scripts/` directory via `./run.sh`.

**Fix:**
- Created symlink: `~/bin/cantoflow` → `scripts/run.sh`
- Added `export PATH="$HOME/bin:$PATH"` to both `~/.bash_profile` and `~/.zprofile`
- Fixed `run.sh` symlink resolution — added `readlink` loop so `PROJECT_DIR` is always resolved from the real script location, not the symlink path

**Usage:** `cantoflow` from any directory. Supports `--funasr` shortcut flag.
