# CantoFlow 工作交接文件 / Handover — 2026-02-18

## 項目背景 / Project Background

**CantoFlow** 是一個 macOS 粵語語音輸入工具，透過按住 Fn 鍵錄音，釋放後自動轉錄並插入文字至當前焦點應用程式。系統支援本地 Whisper 及 FunASR 兩種語音識別後端，並可選配 LLM（Qwen／OpenAI／Anthropic）對轉錄文字進行潤色。

**CantoFlow** is a macOS Cantonese speech-to-text input tool. The user holds the Fn key to record, and upon release the audio is transcribed and inserted into the focused application. It supports two STT backends (local Whisper and FunASR), with optional LLM polishing via Qwen, OpenAI, or Anthropic.

## 一、啟用 Qwen LLM 潤色 / Enable Qwen LLM Polish

### 問題 / Problem
每次錄音的 `polish_status` 均為 `not_run`，LLM 潤色功能未有運作。

Every recording showed `polish_status: not_run`; LLM polishing was never triggered.

### 根本原因 / Root Cause
開發用二進位檔（`.build/release/cantoflow`）從終端機直接執行，並無設定 `QWEN_API_KEY` 環境變數。`polishProvider` 雖預設為 `.auto`，但因找不到任何 API 金鑰而退化為 `.none`。

The development binary was launched directly from the terminal without `QWEN_API_KEY` set. Although `polishProvider` defaults to `.auto`, it falls back to `.none` when no API key is found.

### 修復方案 / Fix
- 建立 `~/.cantoflow.env`（權限 `600`），儲存 `QWEN_API_KEY`
- 建立 `scripts/run.sh`，啟動前自動 `source ~/.cantoflow.env`
- 同時修補 `/Applications/CantoFlow.app/Contents/MacOS/cantoflow`，確保以 .app 形式啟動時亦能讀取金鑰

- Created `~/.cantoflow.env` (permissions `600`) containing `QWEN_API_KEY`
- Created `scripts/run.sh` which sources the env file before launching the binary
- Also patched the `.app` launch script for consistency

### 驗證 / Verified
```
polish_status: ok
provider:      qwen
polish:        ~2,000–3,500 ms
```

## 二、繁體中文輸出修正 / Traditional Chinese Output Fix

### 問題 / Problem
Whisper 使用 `-l yue`（粵語）時，輸出傾向簡體中文。STT 提示語及 LLM 系統提示均未明確要求繁體中文輸出。

Whisper with `-l yue` tends to output Simplified Chinese. Neither the STT prompt nor the LLM system prompt explicitly required Traditional Chinese output.

### 修改檔案 / Files Changed

| 檔案 / File | 修改內容 / Change |
|---|---|
| `Core/Vocabulary/VocabularyStore.swift:336` | Whisper 初始提示加入「請以繁體中文輸出」 |
| `Core/WhisperRunner.swift:101` | 備用提示加入「必須」強調字眼 |
| `Core/TextPolisher.swift` | LLM 系統提示加入第 9 條規則：必須繁體，簡體轉繁體 |

### 備註 / Note
Whisper 的提示語能改善繁體輸出，但並非百分百可靠。LLM 潤色層的第 9 條規則為最終保障。

Whisper prompt hints improve Traditional output but cannot guarantee it 100%. The LLM polish rule 9 is the definitive safeguard.

## 三、終端機重複輸出修復 / Terminal Double-Paste Fix

### 問題 / Problem
在 macOS Terminal 內使用 CantoFlow 時（例如在 Claude Code 命令列輸入），文字出現兩次——原始轉錄文字與潤色後文字先後貼上，並列留存。

When dictating into macOS Terminal (e.g. Claude Code CLI), text appeared twice — the raw transcription and the polished text were both pasted and left concatenated.

### 根本原因 / Root Cause
`fast_ime` 模式下，系統先貼上原始文字，再發送 `Cmd+Z` 撤銷，最後貼上潤色文字。然而 `Cmd+Z` 在終端機中不具備撤銷文字輸入的功能，導致兩段文字同時留存。

In `fast_ime` mode, the system pastes raw text, sends `Cmd+Z` to undo it, then pastes the polished version. In terminals, `Cmd+Z` does not undo text input, so both pastes accumulated.

### 修復方案 / Fix

**`Core/TextInserter.swift`**
- 新增已知終端機 Bundle ID 清單（macOS Terminal、iTerm2、Warp、Kitty、VSCode、Alacritty、Hyper）
- 新增 `isFrontmostAppTerminal() -> Bool` 方法

- Added a set of known terminal bundle IDs
- Added `isFrontmostAppTerminal() -> Bool`

**`Core/STTPipeline.swift`**
- 於錄音停止時即時偵測當前應用程式是否為終端機
- **終端機路徑：** 跳過原始文字貼上，待 LLM 潤色完成後直接貼上潤色文字
- **一般應用程式路徑：** 維持原有行為（貼上原始 → `Cmd+Z` → 貼上潤色）

- Detects terminal at pipeline start (before STT processing delay)
- **Terminal path:** skips raw paste entirely; pastes polished text directly after LLM completes
- **Regular app path:** unchanged (raw paste → `Cmd+Z` undo → paste polished)

### 驗證 / Verified
```
raw_status:     skipped_terminal
replace_status: terminal_paste
```

## 四、全域 `cantoflow` 命令 / Global `cantoflow` Command

### 問題 / Problem
應用程式只能從 `scripts/` 目錄以 `./run.sh` 方式啟動，不便日常使用。

The app could only be launched from the `scripts/` directory via `./run.sh`.

### 修復方案 / Fix
- 建立符號連結：`~/bin/cantoflow` → `scripts/run.sh`
- 將 `export PATH="$HOME/bin:$PATH"` 加入 `~/.bash_profile` 及 `~/.zprofile`（用戶目前使用 bash）
- 修正 `run.sh` 的符號連結解析邏輯——以 `readlink` 迴圈找出真實腳本位置，避免 `PROJECT_DIR` 計算錯誤

- Created symlink: `~/bin/cantoflow` → `scripts/run.sh`
- Added `~/bin` to `PATH` in both `~/.bash_profile` and `~/.zprofile`
- Fixed symlink resolution in `run.sh` using a `readlink` loop

### 用法 / Usage
```bash
cantoflow            # 正常啟動（Whisper fast + Qwen polish）
cantoflow --funasr   # FunASR 模式
```
