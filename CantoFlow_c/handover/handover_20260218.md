# CantoFlow 工作交接文件 / Handover — 2026-02-18

## 項目背景 / Project Background

**CantoFlow** 是一個 macOS 粵語語音輸入工具，透過按住 Fn 鍵錄音，釋放後自動轉錄並插入文字至當前焦點應用程式。系統支援本地 Whisper 及 FunASR 兩種語音識別後端，並可選配 LLM（Qwen／OpenAI／Anthropic）對轉錄文字進行潤色。

**CantoFlow** is a macOS Cantonese speech-to-text input tool. The user holds the Fn key to record, and upon release the audio is transcribed and inserted into the focused application. It supports two STT backends (local Whisper and FunASR), with optional LLM polishing via Qwen, OpenAI, or Anthropic.

## 一、v0.2.2 — Qwen LLM 潤色啟用 / Enable Qwen LLM Polish

### 問題 / Problem
每次錄音的 `polish_status` 均為 `not_run`，LLM 潤色功能未有運作。

Every recording showed `polish_status: not_run`; LLM polishing was never triggered.

### 根本原因 / Root Cause
開發用二進位檔從終端機直接執行，並無設定 `QWEN_API_KEY` 環境變數。`polishProvider` 雖預設為 `.auto`，但因找不到任何 API 金鑰而退化為 `.none`。

The dev binary was launched directly from terminal without `QWEN_API_KEY` set. Although `polishProvider` defaults to `.auto`, it falls back to `.none` when no API key is found.

### 修復 / Fix
- 建立 `~/.cantoflow.env`（權限 `600`），儲存 `QWEN_API_KEY`
- 建立 `scripts/run.sh`，啟動前自動 `source ~/.cantoflow.env`
- 修補 `/Applications/CantoFlow.app/Contents/MacOS/cantoflow` 確保 .app 形式亦能讀取

### 驗證 / Verified
```
polish_status: ok
provider:      qwen
polish:        ~2,000–3,500 ms
```

## 三、v0.2.2 — 終端機重複輸出修復 / Terminal Double-Paste Fix

### 問題 / Problem
在 macOS Terminal（如 Claude Code CLI）輸入時，文字出現兩次。

When dictating into macOS Terminal (Claude Code CLI), text appeared twice.

### 根本原因 / Root Cause
`fast_ime` 模式先貼原始文字 → `Cmd+Z` 撤銷 → 貼潤色文字。但 `Cmd+Z` 在終端機不能撤銷輸入，導致兩段文字並存。

`fast_ime` pastes raw text, then `Cmd+Z` to undo, then pastes polished. But `Cmd+Z` doesn't undo text in terminals.

### 修復 / Fix

**`Core/TextInserter.swift`**
- 新增已知終端機 Bundle ID 清單（Terminal、iTerm2、Warp、Kitty、VSCode、Alacritty、Hyper）
- 新增 `isFrontmostAppTerminal() -> Bool`

**`Core/STTPipeline.swift`**
- 偵測到終端機時，跳過原始貼上，直接等 LLM 潤色後再貼（`terminal_paste` 路徑）

### 驗證 / Verified
```
raw_status:     skipped_terminal
replace_status: terminal_paste
```
