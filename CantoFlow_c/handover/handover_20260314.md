## 一、Fresh Clone 可用化 / Fresh Clone Bootstrap

### 問題 / Problem
原 repo `git clone` 後即使 `swift build` 成功，仍然缺少 `whisper.cpp`、模型檔與完整安裝步驟，導致 app 可開但無法 STT。

### 修復 / Fix
- 擴充 `install.sh`
  - 檢查/安裝 Homebrew 依賴：`cmake`、`ffmpeg`、`jq`
  - 自動 clone `third_party/whisper.cpp`
  - 自動編譯 `whisper-cli`
  - 自動下載 `large-v3-turbo`、`large-v3`、`small` 模型
  - 自動建立 `~/bin/cantoflow`
  - 自動建立 `~/.cantoflow.env`
- 更新 `README.md`、`INSTALL.md`

### 影響 / Impact
fresh clone 後只需執行：
```bash
bash install.sh
```
即可完成本地 STT 依賴建置。

## 二、啟動前健康檢查 / Startup Health Checks

### 問題 / Problem
用戶難以分辨「冇上字」到底係因為權限、模型、hotkey listener、定 input device 問題。

### 修復 / Fix
於 `AppDelegate.swift` 新增啟動檢查：
- Microphone permission
- Accessibility
- Hotkey listener / Input Monitoring
- `whisper-cli` 是否存在
- model 是否存在
- 是否有音訊輸入裝置
- 目前 input device 名稱

### 影響 / Impact
啟動通知及 terminal log 會更清楚指出缺失資產或缺失權限。

## 三、可選錄音裝置 / Selectable Input Device

### 問題 / Problem
原本 `audioDevice` 設定並未真正套用，CantoFlow 永遠依賴 macOS 系統預設 microphone。

### 修復 / Fix
- 新增 `Core/AudioDeviceManager.swift`
- 以 CoreAudio / AudioUnit 在錄音開始前切換 input device
- `General` 頁面新增 `Input Device` 選單
- Menu Bar 顯示目前 input device

### 影響 / Impact
用戶可明確指定 microphone，避免誤用 webcam / continuity mic，直接提升 STT 準確率。

## 四、Launch at Login 真正可用 / Working Launch at Login

### 問題 / Problem
`Launch at Login` toggle 原本被硬編碼 disable，只是 UI 假開關。

### 修復 / Fix

## 五、API Keys UX 與 Gemini 支援 / API Key UX and Gemini Support

### 問題 / Problem
- API key 欄位曾無法穩定輸入
- 已輸入 key 會明文顯示
- Gemini endpoint 測試回 404

### 修復 / Fix
- `API Keys` 頁面改成：
  - 按 Enter 後由明文編輯狀態切回部分遮罩
  - 顯示首尾四字，中間以圓點 redact
  - 加入單一 `Test API Key Endpoint` 按鈕與紅綠燈
- 新增 Gemini polish support
  - `TextPolisher.swift` 加入 `gemini` provider
  - `AppConfig.swift` 加入 `gemini` provider CLI help
  - UI 改為 `Google Gemini`、`Qwen / DashScope`、`OpenAI`
  - 移除 Anthropic UI

### Gemini Debug 結果 / Gemini Debug Result
使用 workspace `.env` 中 `GEMINI_API_KEY` 以 `curl` 驗證：
- `gemini-2.0-flash` → `404`
- `gemini-2.5-flash` → `200`

### 根因 / Root Cause
Google 返回明確錯誤：`gemini-2.0-flash` 已不再向新用戶開放。

### 最終修復 / Final Fix
預設 Gemini model 改為：
```text
gemini-2.5-flash
```

## 六、Vocabulary UI 重構 / Vocabulary UI Refactor

### 問題 / Problem
原本 personal vocabulary 只係單一清單，難以分類管理；同時在 macOS 下偶爾觸發 `NSTableView delegate reentrant operation` warning。

### 修復 / Fix
- Vocabulary 改成分類 TAB：
  - `全部`
  - `地名`
  - `動作`
  - `口頭禪`
  - `食物`
  - `公司`
  - `技術`
  - `其他`
- 將 `List(selection:)` 改為明確 click-open 編輯，減少 AppKit reentrant 風險
- import/export/reload 後的 UI 更新延後到下一個 main-loop tick

### 影響 / Impact
Vocabulary 使用體驗更接近真正分類詞庫；已降低部份 `NSTableView` reentrant warning 觸發點，但未保證 100% 消失。
